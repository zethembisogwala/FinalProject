Computer Security is a method of using defense protocals to protect computer systems and networks. When an unauthorized person tries to get into a computer system – “computer security” is used in detecting that person and preventing that person from getting acces to it. In order to be safe in todays world computer security is very essential. The main objective is to protect companies important & private data from theft.

There are three layers in the computer security:
  • Hacking An unauthorized person uses the computer is known as hacking. Cyber crimes main components include hacking.
  • Cracking When the intention of the unauthorized person is of causing harm and creating problem than such a unauthorized intrusion is known as cracking.
  • Phreaking Cracking of the phone network by unauthorized person it is known as phreaking.

Measures taken for the improvement in computer security are:

  • The unauthorized persons who would potentially compromise security not be allowed to access computer systems.
  • Dependence on computer programs for computer security is prohibited and hence necessity for development of hardware mechanism.
  • Operating system mechanism development is also necessary for imposing the rules on programs.
  • Planning for programming strategies is also very important.

There are many ways for doing this. Here are various ways:
  • One of the most common ways of computer security is using firewall. It can be setup to let only authenticated data flow in or out of the network.

Computer securities have some technical areas which are:
  • Confidentiality
  • Integrity
  • Authentication

Confidentiality here means keeping things private or in secrecy so that unauthorized person cannot use it then comes integrity which means changes cannot be made by any unauthorized user as it is protected and last authentication means the person who is authorized for that that.

Computer security professionals are concerned mainly about access control and nonrepudiation.

Access control means not only that user can access the resource and services but user has access to the resources which he legitimately expects.

Nonrepudiation means if a person sends a message then he cannot deny that he has sent it and vice versa.
